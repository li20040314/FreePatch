# FreePatch 补丁打包工具 — 操作说明与架构文档

> 版本：1.0.0 | 技术栈：Rust + Axum + 原生 HTML/JS

---

## 一、项目简介

FreePatch 是一个 Java Web 项目补丁打包工具，通过 **浏览器 Web UI** 提供图形化操作界面。用户只需指定项目路径和补丁文件列表，工具即可自动将补丁涉及的源文件打包为标准的 **Java Web 补丁包目录结构**（`WEB-INF/classes/` + 静态资源），可直接用于生产环境部署。

### 核心功能

| 功能 | 说明 |
|------|------|
| **补丁打包** | 读取补丁文件 → 解析文件列表 → 按规则映射源/目标路径 → 复制到输出目录 |
| **配置管理** | 多配置文件（Profile）支持，可保存/切换/增删配置项 |
| **补丁文件编辑** | 内嵌编辑器，可直接在浏览器中查看和修改 `diff.txt` |
| **实时日志** | 基于 SSE（Server-Sent Events）的实时日志流显示 |
| **路径自动推导** | 输入项目路径后，自动填充 class 路径、导出路径、版本号等 |
| **原生文件对话框** | 使用操作系统原生文件/文件夹选择器 |

## 二、项目架构

### 2.1 目录结构

```
FreePatchGUI/rust/
├── Cargo.toml              # 项目配置与依赖
├── Cargo.lock              # 依赖锁定文件
├── rustup-init.exe         # Rust 安装工具（可选）
├── 启动工具.bat            # Windows 一键启动脚本
├── src/
│   ├── main.rs             # 入口点 + AppState 共享状态
│   ├── config.rs           # 配置管理：Settings、ProfileStore、JSON 读写
│   ├── server.rs           # HTTP API 服务：Axum 路由 + SSE + 文件对话框
│   ├── patch.rs            # 核心补丁引擎：路径解析 + 文件复制
│   └── index.html          # 前端 UI（嵌入二进制中）
├── target/
│   └── release/
│       └── free-patch-gui.exe  # 编译后的可执行文件
└── docs/
    └── 操作说明与架构文档.md  # 本文档
```

### 2.2 模块职责

| 模块 | 文件 | 职责 |
|------|------|------|
| **入口** | `main.rs` | 初始化配置、创建共享状态 `AppState`、启动 HTTP 服务 |
| **配置** | `config.rs` | `Settings`（单配置）、`ProfileStore`（多配置集合）的定义、JSON 序列化/反序列化、文件读写 |
| **服务器** | `server.rs` | Axum 路由注册、请求处理、SSE 日志流、原生文件对话框、补丁启动编排 |
| **补丁引擎** | `patch.rs` | 补丁文件解析、路径映射（6 种策略）、文件复制、进度回调 |
| **前端** | `index.html` | 单页 Web UI（内联 CSS/JS），通过 HTTP API 与后端交互 |

### 2.3 数据流架构

```

│  │ (配置 I/O) │   │  (HTTP API)  │    │  (补丁引擎/阻塞线程)  │   │
│  └──────────┘    └──────────────┘    └──────────────────────┘   │
│       ▲                 │ ▲                      │               │
│       │                 │ │                      │               │
│  ┌────┴─────────────────┴─┴──────────────────────┘               │
│  │                     AppState (Arc)                            │
│  │  ┌─────────────────────┐  ┌─────────────────────────────┐    │
│  │  │ config: Mutex<AppConfig> │  log_state: Mutex<LogState>    │
│  │  └─────────────────────┘  └─────────────────────────────┘    │
│  └──────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────┘
           │                        ▲
           │ HTTP (localhost:18080) │ SSE 实时日志

┌───────────────────────────────────────────────────────┐
│                    浏览器 (Web UI)                     │
│   原生 HTML/CSS/JS · 配置表单 · 补丁编辑器 · 日志面板   │
│   通过 fetch() 调用 REST API · EventSource 接收 SSE    │
└───────────────────────────────────────────────────────┘
```

### 2.4 启动流程

1. `main()` 获取可执行文件所在目录，拼接 `config.json` 路径
2. 调用 `config::load_config()` 加载配置（支持向后兼容）
3. 创建 `Arc<AppState>` 作为全局共享状态
4. 调用 `server::run_server()`：
   - 注册 10 条 HTTP 路由
   - 在 `18080~18090` 范围扫描可用端口
   - 自动打开默认浏览器
   - 启动 axum HTTP 服务器

### 2.5 并发模型

| 机制 | 用途 |
|------|------|
| `tokio::sync::Mutex` | 保护 `AppState.config` 和 `AppState.log_state` 的并发访问 |
| `tokio::sync::broadcast` | 日志发布-订阅架构：补丁线程发布消息，SSE 端点订阅 |
| `tokio::task::spawn_blocking` | 运行阻塞操作：补丁文件复制、原生文件对话框 |
| 单线程 Tokio 运行时 | 默认 `#[tokio::main]`（current_thread 模式） |

### 2.6 依赖说明

| 依赖 | 用途 |
|------|------|
| **axum 0.7** | HTTP 路由框架，提供提取器、SSE 响应、共享状态 |
| **tokio 1** | 异步运行时、Mutex、broadcast channel、spawn_blocking |
| **serde / serde_json** | 配置和 API 的 JSON 序列化/反序列化 |
| **tower-http** | CORS 中间件，允许浏览器跨域请求 |
| **chrono** | 日志时间戳、版本号日期生成 |
| **futures / tokio-stream** | SSE 流处理：迭代旧日志 + 广播实时日志 |
| **rfd** | 原生操作系统文件/文件夹选择对话框 |

## 三、环境准备与构建

### 3.1 环境要求

- **Rust 工具链**（1.70+，推荐使用 [rustup](https://rustup.rs/) 安装）
- Windows 7+（当前仅支持 Windows，因 `rfd` 对话框和 `.bat` 启动脚本）
- 浏览器（Chrome/Edge/Firefox 等现代浏览器）

### 3.2 安装 Rust

方式一：使用项目附带的 `rustup-init.exe`（双击运行，按提示安装）

方式二：从 [https://rustup.rs](https://rustup.rs) 下载安装

安装完成后，重启终端验证：

```bash
rustc --version
cargo --version
```

### 3.3 构建项目

```bash
# 进入项目目录
cd FreePatchGUI/rust

# 发布构建（优化体积）
cargo build --release
```

构建产物：`target/release/free-patch-gui.exe`（约 5~8 MB）

### 3.4 运行

**方式一：双击启动脚本**

双击 `启动工具.bat`，脚本会自动查找 `free-patch-gui.exe` 并启动。

**方式二：直接运行**

```bash
# 方法 A：使用 cargo
cargo run --release

# 方法 B：直接执行
.\target\release\free-patch-gui.exe
```

启动后命令行输出：

```
FreePatch 补丁打包工具已启动
请访问 http://localhost:18080
```

浏览器将自动打开操作界面。

### 3.5 构建配置说明（Cargo.toml）

```toml
[profile.release]
opt-level = "z"   # 优化二进制体积
lto = true        # 链接时优化（减小体积 + 提升性能）
strip = true      # 去除调试符号（减小体积）
```

## 四、操作说明

### 4.1 主界面概览

打开浏览器访问 `http://localhost:18080`，界面分为四个区域：

1. **配置管理栏**（顶部）— 配置文件切换、新建、保存、删除
2. **参数设置区**（中间）— 项目路径、补丁文件、版本号等参数
3. **操作按钮** — 开始打包
4. **日志面板**（底部）— 实时显示打包进度和结果

### 4.2 配置管理（Profile）

工具支持多配置（Profile），以 `config.json` 文件保存在可执行文件同目录。

**操作步骤：**

| 操作 | 说明 |
|------|------|
| **切换配置** | 从下拉列表选择已有配置，自动填充表单 |
| **新建配置** | 点击「新建配置」，输入配置名称，当前表单保存为新配置 |
| **保存配置** | 修改参数后点击「保存配置」，覆盖当前配置 |
| **删除配置** | 选中要删除的配置，点击「删除配置」（不可删除最后一个配置） |

> 配置会自动保存到 `config.json`，下次启动自动加载。

### 4.3 参数设置

| 参数 | 必填 | 说明 | 自动推导 |
|------|------|------|----------|
| **项目路径** | ✅ | Java Web 项目的根目录 | — |
| **补丁文件** | ✅ | `diff.txt` 补丁文件路径 | `{project}\diff.txt` |
| **版本号** | ❌ | 输出目录的版本标识 | `{dirname}-{yyyyMMdd-HHmmss}` |
| Class 目录 | ❌ | 编译后的 class 文件目录 | `{project}\target\classes` |
| 导出目录 | ❌ | 补丁包输出目录 | `{project}\target\patch_pkg` |
| src/java 目录 | ❌ | Java 源码前缀 | `src/main/java` |
| src/resources 目录 | ❌ | 资源文件前缀 | `src/main/resources` |
| src/webapp 目录 | ❌ | Web 资源前缀 | `src/main/webapp` |
| WebContent | ❌ | Eclipse 旧项目 Web 目录 | `WebContent` |

> 填写项目路径后，其余字段会自动推导。如需自定义，可手动修改。

### 4.4 补丁打包流程

1. **填写项目路径**（必填）— 选择 Java Web 项目根目录
2. **指定补丁文件**（必填）— 选择或创建 `diff.txt`
3. **确认版本号** — 按需修改，默认包含项目名和时间戳
4. 展开 **「高级设置」** 按需调整路径映射规则
5. 点击 **「开始打包」**
6. 在 **日志面板** 查看实时进度
7. 打包完成后，在导出目录获取补丁包

### 4.5 补丁文件编辑器

点击补丁文件旁的 **「编辑」** 按钮，可在浏览器中直接编辑补丁文件：

- 支持标准格式（每行一个路径）和 SVN Eclipse 格式（`Index: 路径`）
- 编辑完成后点击 **「保存」** 将内容写回磁盘
- 点击 **「取消」** 关闭编辑器，不保存

### 4.6 补丁文件格式

工具支持两种补丁文件格式，可在同一文件中混用：

**标准格式**（每行一个相对路径）：
```
com/example/UserService.java
src/main/webapp/index.jsp
src/main/resources/log4j.properties
static/js/app.js
```

**SVN Eclipse 插件格式**：
```
Index: com/example/UserService.java
Index: src/main/webapp/index.jsp
```

## 五、路径解析策略（核心逻辑）

工具根据补丁文件中的路径前缀，智能映射到源文件和目标文件位置。

### 路径映射规则一览

| 优先级 | 路径前缀 | 源文件位置 | 目标文件位置 |
|--------|----------|-----------|------------|
| 1 | `src/main/java/...` | `{classPath}/{file}` (`.java` → `.class`) | `{desPath}/{version}/WEB-INF/classes/{file}.class` |
| 2 | `src/main/resources/...` | `{classPath}/{file}` | `{desPath}/{version}/WEB-INF/classes/{file}` |
| 3 | `src/main/webapp/...` | `{projectPath}/{fullPath}` | `{desPath}/{version}/{relative}`（去掉 webapp 前缀） |
| 4 | `src\...`（旧版 Eclipse） | 同规则 1 (`.java` → `.class`) | `{desPath}/{version}/WEB-INF/classes/{file}.class` |
| 5 | `WebContent\...`（旧版） | `{projectPath}/{fullPath}` | `{desPath}/{version}/{relative}`（去掉 WebContent 前缀） |
| 6 | 回退规则 | `{projectPath}/{path}` | `{desPath}/{version}/{path}` |

### 输出目录示例

```
target/patch_pkg/
└── myproject-20260528-180951/      ← 版本号
    ├── WEB-INF/
    │   └── classes/
    │       ├── com/example/        ← Java 类文件（.class）
    │       ├── log4j.properties    ← resources 文件
    │       └── ...
    ├── index.jsp                   ← Web 资源
    ├── static/
    │   └── js/app.js
    └── ...
```

## 六、API 接口文档

后端 HTTP 服务提供以下 REST API：

### 6.1 页面

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 返回嵌入的 `index.html` 单页应用 |

### 6.2 配置管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/config` | 获取当前配置（`ProfileStore` JSON） |
| POST | `/api/config` | 保存配置（接收 `ProfileStore` JSON 正文） |

**POST `/api/config` 请求体示例：**

```json
{
  "currentProfile": "生产环境",
  "profiles": {
    "生产环境": {
      "patchFile": "D:/project/diff.txt",
      "projectPath": "D:/project",
      "webContent": "WebContent",

      "desPath": "D:/project/target/patch_pkg",
      "version": "myproject-20260528-180951",
      "srcJavaPrefix": "src/main/java",
      "srcResourcePrefix": "src/main/resources",
      "srcWebappPrefix": "src/main/webapp"
    }
  }
}
```

### 6.3 补丁操作

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/start` | 启动补丁打包（接收 `Settings` JSON） |
| GET | `/api/logs` | SSE 实时日志流 |

**POST `/api/start` 请求体示例：**

```json
{
  "patchFile": "D:/project/diff.txt",
  "projectPath": "D:/project",
  "version": "myproject-20260528-180951",
  "classPath": "D:/project/target/classes",
  "desPath": "D:/project/target/patch_pkg",
  "srcJavaPrefix": "src/main/java",
  "srcResourcePrefix": "src/main/resources",
  "srcWebappPrefix": "src/main/webapp",
  "webContent": "WebContent"
}
```

### 6.4 补丁文件编辑

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/patch-content?path=...` | 读取补丁文件内容 |
| POST | `/api/patch-content` | 保存补丁文件内容 |
| POST | `/api/ensure-file` | 确保补丁文件存在（不存在则创建空文件） |

**POST `/api/patch-content` 请求体：**

```json
{
  "path": "D:/project/diff.txt",
  "content": "com/example/UserService.java\nsrc/main/webapp/index.jsp"
}
```

### 6.5 文件对话框

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/pick-file` | 打开原生文件选择器，返回文件路径 |
| POST | `/api/pick-folder` | 打开原生文件夹选择器，返回文件夹路径 |

### 6.6 响应格式

**成功响应：**
```json
{ "status": "ok" }
```

**失败响应（HTTP 400）：**
```json
{ "status": "error", "message": "错误描述" }
```

## 七、常见问题

### Q1: 启动后浏览器无法自动打开？

手动访问控制台输出的地址，通常是 `http://localhost:18080`。

### Q2: 端口 18080 被占用？

工具会自动尝试 `18080~18090` 范围的端口，直到找到可用端口。如果全部被占用，启动会失败，请关闭占用端口的程序后重试。

### Q3: 提示 "找不到 free-patch-gui.exe"？

先执行 `cargo build --release` 编译，或在项目根目录放置编译好的 `free-patch-gui.exe`。

### Q4: 打包结果在哪里？

默认在 `{项目路径}/target/patch_pkg/{版本号}/` 目录。

### Q5: 如何自定义输出目录？

在参数设置中修改 **「导出目录」** 字段（在高级设置中）。

### Q6: 配置文件在哪里？

在 `free-patch-gui.exe` 同目录下的 `config.json`。可手动编辑，但建议通过 UI 操作。

### Q7: 补丁文件中可以使用绝对路径吗？

可以。如果补丁文件中的路径以项目路径开头，工具会自动提取相对部分进行处理。

## 八、开发扩展指南

### 添加新的 API 路由

在 `src/server.rs` 中：

1. 编写处理函数（如 `async fn handle_my_api()`）
2. 在 `run_server()` 的 `Router::new()` 链中添加 `.route("/api/my-api", get(handle_my_api))`
3. 在前端 `index.html` 中通过 `fetch('/api/my-api')` 调用

### 添加新的路径解析策略

在 `src/patch.rs` 的 `resolve_path()` 函数中添加新的策略分支：

```rust
// 示例：新增 Thymeleaf 模板目录支持
if n.starts_with("src\\main\\templates\\") {
    let rel = &n["src\\main\\templates\\".len()..];
    let src = PathBuf::from(&s.project_path).join(&n);
    let dst = PathBuf::from(&s.des_path).join(&s.version).join("WEB-INF").join("templates").join(rel);
    return (src, dst);
}
```

### 构建体积优化

当前 release 配置已使用 `opt-level = "z"` + `lto = true` + `strip = true`。如需进一步减小体积：

```toml
[profile.release]
opt-level = "z"
lto = true
strip = true
codegen-units = 1  # 合并代码生成单元，进一步优化体积
panic = "abort"    # 不使用 panic 回溯，减少体积
```

### 发布新版本

1. 修改 `Cargo.toml` 中的 `version` 字段
2. 执行 `cargo build --release`
3. 分发 `target/release/free-patch-gui.exe`（无需任何其他文件）

## 九、技术要点

- **单二进制分发**：前端 `index.html` 通过 `include_str!()` 宏在编译时嵌入二进制文件，运行时无需额外文件
- **无数据库**：所有状态存于内存（`Arc<AppState>`）或 `config.json`
- **无 TLS/认证**：仅绑定 `0.0.0.0`，仅限本地使用
- **向后兼容配置**：`load_config()` 支持从新格式（`ProfileStore`）→ 旧格式（单个 `Settings`）→ 默认配置的三级回退
- **端口自动扫描**：`bind_available()` 在指定范围内查找第一个可用端口
- **无测试代码**：当前项目不包含单元测试或集成测试（`tests/` 目录和 `#[cfg(test)]` 模块均不存在）
